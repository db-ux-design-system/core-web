# @db-ux/core-vite-plugin

Vite plugin that generates an optimized CSS bundle for DB UX Design System — automatically including only the component styles, tokens, and features actually used in the project.

## Key Facts

- **ESM only** (`"type": "module"`)
- Built with `tsc` to `build/`
- Peer dependency: `vite >=4.0.0`
- Entry CSS placeholder: `index.css` (replaced at build/dev time by the plugin)

## Scripts

```bash
pnpm run build        # Compile TypeScript
pnpm run test         # Run vitest
pnpm run test:update  # Update vitest snapshots
```

## Structure

```text
src/           # Plugin source (TypeScript)
test/          # Vitest tests
index.css      # Placeholder CSS entry — published as-is, transformed by the plugin at runtime
build/         # Compiled output — never commit
```

## How It Works

1. Detects DB UX component usage in source files (e.g. `class="db-button"`)
2. Detects design token usage
3. Generates an optimized CSS bundle with only the required styles
4. Replaces `@import "@db-ux/core-vite-plugin/index.css"` with the generated bundle

## Plugin Options

| Option             | Default     | Description                                                        |
| ------------------ | ----------- | ------------------------------------------------------------------ |
| `include`          | auto-detect | Force-include specific components, foundations, colors, densities  |
| `exclude`          | none        | Exclude specific components, foundations, colors, densities        |
| `optimize`         | `true`      | Remove unused CSS variable declarations                            |
| `theme`            | auto-detect | Preferred theme package name (e.g. `"db-theme"`)                  |
| `additionalLayers` | none        | Append custom layers before/after the auto-generated layer order   |
| `overrideLayers`   | none        | Fully replace the auto-generated `@layer` statement                |
| `debug`            | `false`     | Write `db-ux-detection-report.json` for troubleshooting            |

## Important Notes

- **Plugin order matters with Tailwind**: `dbUxPlugin()` must come **before** `tailwindcss()` in `vite.config.ts`
- **CSS must be imported via JS/TS** (`import "./styles.css"` in `main.ts`), not via `<link>` tags — static link tags bypass Vite's module graph and the plugin won't transform them
- **Vite 8+**: Set `css.transformer: 'postcss'` if using PostCSS plugins alongside this plugin, since Vite 8 defaults to LightningCSS
- Never commit `build/` — it is generated by `tsc`
