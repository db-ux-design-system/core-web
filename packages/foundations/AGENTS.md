# @db-ux/core-foundations

Provides all design tokens (colors, spacing, typography, elevation, density) and assets (icons, fonts) for the DB UX Design System. This is the base layer that `@db-ux/core-components` depends on.

## Key Facts

- **ESM only** (`"type": "module"`)
- Tokens are authored in SCSS (`scss/`) and compiled to CSS, SCSS variables, and Tailwind tokens
- Assets (icons, fonts) live in `assets/`
- Build output goes to `build/`

## Scripts

```bash
pnpm run build              # Full build: normalize → copy → Sass → Tailwind → PostCSS → IDE → TS
pnpm run dev                # Prepare + start Vite dev server
pnpm run test:e2e           # Run Playwright e2e tests (do NOT run regenerate:screenshots locally)
```

## Structure

```text
scss/           # SCSS source for all tokens and helpers
assets/         # Icons (woff2) and fonts
tailwind/       # Tailwind token config
ide/            # IDE integration files (e.g. VS Code snippets)
src/            # TypeScript source (icon names, type exports)
build/          # Compiled output — never commit
```

## Token Categories

- `colors` — adaptive color system (`--db-color-*`)
- `spacing` — fixed and responsive spacing (`--db-spacing-*`)
- `typography` — font sizes, line heights, font families
- `elevation` — box-shadow levels
- `density` — functional / regular / expressive scaling

## Changeset Rules

Changes in `packages/foundations/scss` require a changeset for `@db-ux/core-foundations`.

- `patch` — bug fix
- `minor` — new token or feature
- `major` — removed or renamed token (breaking for consumers)

## Important Notes

- **Never hardcode values** — always use `var(--db-*)` tokens in component styles
- **Never commit `build/`** — it is generated by the build script
- The `assets/` folder is committed and published; changes there affect all consumers
- `regenerate:screenshots` must only run in CI/CD — never locally
