# Migration (4.x.x) ➡ 5.0.0

## Refactoring of Header and Navigation

We encountered some problems with the current `DBHeader` as well with the `DBNavigation`.
By splitting the `DBHeader` into 2 components (`DBControlPanelDesktop` & `DBControlPanelMobile`),
we are able to control our navigation based on the device size.
Additionally, we split the `DBNavigationItem` as well to provide a new `DBNavigationItemGroup` instead of
having a Slot with `subNavigation`.

For a detailed migration based on framework checkout those:

- [Angular](400-500/shell/Angular.md)
- [React](400-500/shell/React.md)
- [Vue](400-500/shell/Vue.md)

### Changes in DBPage ➡️ DBShell

- **Mandatory Usage**: The `DBShell` component is now mandatory and replaces the optional `DBPage`.
- **Removed Properties**: The following properties from `DBPage` are no longer supported in `DBShell`:
    - `documentOverflow`
    - `variant`
    - `footer`
- **Slot Changes**:
    - The `header` slot has been split into two new slots:
        - `controlPanelDesktop`
        - `controlPanelMobile`

### Changes in DBHeader ➡️ DBControlPanelDesktop/DBControlPanelMobile

- **Component Split**: The `DBHeader` has been split into two components:
    - `DBControlPanelDesktop`
    - `DBControlPanelMobile`
- **Slot Wrapping**: The following slots now require additional wrapping components:
    - `brand` ➡️ `DBControlPanelBrand`
    - `metaNavigation` ➡️ `DBControlPanelMetaNavigation`
    - `primaryActions` ➡️ `DBControlPanelPrimaryActions`
    - `secondaryActions` ➡️ `DBControlPanelSecondaryActions`

### Changes in DBNavigationItem ➡️ DBControlPanelNavigationItemGroup

- **Slot Removal**: The `subNavigation` slot in `DBNavigationItem` has been removed.
- **New Component**: Use the new `DBControlPanelNavigationItemGroup` component to group navigation items instead of relying on the `subNavigation` slot.

#### Example Migration

Before:

```html
<DBNavigationItem><a>Item</a></DBNavigationItem>
<DBNavigationItem>
	Group
	<template slot:sub-navigation>
		<DBNavigationItem><a>Sub Item 1</a></DBNavigationItem>
		<DBNavigationItem><a>Sub Item 2</a></DBNavigationItem>
	</template>
</DBNavigationItem>
```

After:

```html
<DBControlPanelNavigationItem><a>Item</a></DBControlPanelNavigationItem>
<DBControlPanelNavigationItemGroup text="Group">
	<DBControlPanelNavigationItem
		><a>Sub Item 1</a></DBControlPanelNavigationItem
	>
	<DBControlPanelNavigationItem
		><a>Sub Item 2</a></DBControlPanelNavigationItem
	>
</DBControlPanelNavigationItemGroup>
```

### Changes in ESLint Plugin Rules

The following ESLint plugin rule names have been renamed to match the new component names:

| Old Rule Name                                                        | New Rule Name                                                                            |
| -------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| `header-burger-menu-label-required`                                  | `control-panel-mobile-burger-menu-label-required`                                        |
| `navigation-item-back-button-text-required` (for `DBNavigationItem`) | `navigation-item-back-button-text-required` (now targets `DBControlPanelNavigationItem`) |

Update your ESLint configuration if you reference these rules individually.
