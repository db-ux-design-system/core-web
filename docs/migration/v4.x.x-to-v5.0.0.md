# Migration (4.x.x) ➡ 5.0.0

## Tabs: Refactored from radio-input to button-based WAI-ARIA tabs pattern

The entire Tabs component family (`DBTabs`, `DBTabItem`, `DBTabList`, `DBTabPanel`) has been refactored to use native `<button role="tab">` elements instead of the previous `<input type="radio">` + `<label>` approach.

### Removed props

| Before (v4.x)           | Status |    After (v5.x)    | Description                                                                                               |
| ----------------------- | :----: | :----------------: | --------------------------------------------------------------------------------------------------------- |
| `checked` on DBTabItem  |   ❌   |         ❌         | Active state now managed by parent DBTabs                                                                 |
| `noText` on DBTabItem   |   ❌   |         ❌         | Removed — for icon-only tabs omit `label`/children and pass `aria-label` for the accessible name          |
| `name` on DBTabItem     |   ❌   |         ❌         | Managed internally by DBTabs                                                                              |
| `onChange` on DBTabItem |   ❌   |         ❌         | Removed — move per-tab change logic to `onIndexChange` / `onValueChange` on DBTabs                        |
| `change` on DBTabItem   |   ❌   |         ❌         | Removed — move per-tab change logic to `indexChange` / `valueChange` on DBTabs (Angular/Vue/WC)           |
| `name` on DBTabs        |   🔁   |   `id` / `label`   | ID generation is now internal — use `id` for stable IDs/deep links; `label` sets the tablist `aria-label` |
| `onTabSelect` on DBTabs |   🔁   |  `onIndexChange`   | Callback now receives the active index                                                                    |
| `tabSelect` on DBTabs   |   🔁   |   `indexChange`    | Angular/Vue/WC event, now receives the active index                                                       |
| `alignment` on DBTabs   |   🔁   | `tabItemAlignment` | Renamed for clarity                                                                                       |
| `width` on DBTabs       |   🔁   |   `tabItemWidth`   | Renamed for clarity                                                                                       |

### Structural HTML changes

- Tab items: `<label><input type="radio" ...></label>` → `<button role="tab" type="button">`
- Tab panels: `<section>` → `<div role="tabpanel">`
- Tab list: `<ul>` wrapper removed — `<div role="tablist">` is now the direct container

### Behavior changes

<!-- markdownlint-disable MD033 -->

- **Tab panels are now keyboard focusable.** The active `tabpanel` receives `tabindex="0"` (inactive panels get `tabindex="-1"`), following the [WAI-ARIA APG tabs pattern](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/) so keyboard users can reach panel content that has no focusable element of its own. Previously the panel (`<section role="tabpanel">`) was not part of the tab order. If you relied on the old behavior, note that pressing <kbd>Tab</kbd> after the tab list now stops on the active panel.

<!-- markdownlint-enable MD033 -->

### Removed type exports

- **`OverflowScrollButtonProps`** has been removed from `@db-ux/core-components` shared model. Its props have been moved into `DBTabsProps` directly and renamed: `scrollLeftText` → `scrollStartLabel`, `scrollRightText` → `scrollEndLabel` (see table above). `arrowScrollDistance` keeps its name.
- **`OverflowScrollButtonState`** has been removed from `@db-ux/core-components` shared model. The scroll state is now internal to DBTabs.
