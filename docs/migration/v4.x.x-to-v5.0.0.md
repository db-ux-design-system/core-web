# Migration (4.x.x) ➡ 5.0.0

## Tabs: Refactored from radio-input to button-based WAI-ARIA tabs pattern

The entire Tabs component family (`DBTabs`, `DBTabItem`, `DBTabList`, `DBTabPanel`) has been refactored to use native `<button role="tab">` elements instead of the previous `<input type="radio">` + `<label>` approach.

### Removed props

<!-- markdownlint-disable MD060 -->

| Before (v4.x)           | Status |     After (v5.x)   | Description                                        |
| ----------------------- | :----: | :----------------: | -------------------------------------------------- |
| `checked` on DBTabItem  |   ❌   |         ❌         | Active state now managed by parent DBTabs          |
| `noText` on DBTabItem   |   ❌   |         ❌         | Removed — use icon-only tabs with `label` for a11y |
| `name` on DBTabItem     |   ❌   |         ❌         | Managed internally by DBTabs                       |
| `onTabSelect` on DBTabs |   🔁   |  `onIndexChange`   | Callback now receives the active index             |
| `alignment` on DBTabs   |   🔁   | `tabItemAlignment` | Renamed for clarity                                |
| `width` on DBTabs       |   🔁   |   `tabItemWidth`   | Renamed for clarity                                |

<!-- markdownlint-enable MD060 -->

### Renamed data attributes

<!-- markdownlint-disable MD060 -->

| Before           | After                     |
| ---------------- | ------------------------- |
| `data-alignment` | `data-tab-item-alignment` |
| `data-width`     | `data-tab-item-width`     |

<!-- markdownlint-enable MD060 -->

### Structural HTML changes

- Tab items: `<label><input type="radio" ...></label>` → `<button role="tab" type="button">`
- Tab panels: `<section>` → `<div role="tabpanel">`
- Tab list: `<ul>` wrapper removed — `<div role="tablist">` is now the direct container

### Removed type exports

- **`AlignmentProps`** has been removed from `@db-ux/core-components` shared model. Use `AlignmentType` (still exported) to build your own props type, or use `TabItemAlignmentProps` from `tabs/model.ts`.

## Automate migration via CLI

We provide a CLI tool to auto migrate your source code. Use this command in your repository:

```shell
npx @db-ux/core-migration --type=v400_v500 --src=./src
```

Please check the changes made in your codebase afterwards, as this is mainly a simple search & replace and there might be unexpected changes of similar wordings to our properties for any other methods, or further code occurrences that don't even refer to properties.
