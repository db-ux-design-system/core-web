# Migration (4.x.x) ➡ 5.0.0

## Tabs: Refactored from radio-input to button-based WAI-ARIA tabs pattern

The entire Tabs component family (`DBTabs`, `DBTabItem`, `DBTabList`, `DBTabPanel`) has been refactored to use native `<button role="tab">` elements instead of the previous `<input type="radio">` + `<label>` approach.

### Removed props

<!-- markdownlint-disable MD060 -->

| Before (v4.x)           | Status |    After (v5.x)    | Description                                                                                               |
| ----------------------- | :----: | :----------------: | --------------------------------------------------------------------------------------------------------- |
| `checked` on DBTabItem  |   ❌   |         ❌         | Active state now managed by parent DBTabs                                                                 |
| `noText` on DBTabItem   |   ❌   |         ❌         | Removed — the text should be always visible                                                               |
| `name` on DBTabItem     |   ❌   |         ❌         | Managed internally by DBTabs                                                                              |
| `onChange` on DBTabItem |   ❌   |         ❌         | Removed — move per-tab change logic to `onIndexChange` / `onValueChange` on DBTabs                        |
| `change` on DBTabItem   |   ❌   |         ❌         | Removed — move per-tab change logic to `indexChange` / `valueChange` on DBTabs (Angular/Vue/WC)           |
| `name` on DBTabs        |   🔁   |   `id` / `label`   | ID generation is now internal — use `id` for stable IDs/deep links; `label` sets the tablist `aria-label` |
| `onTabSelect` on DBTabs |   🔁   |  `onIndexChange`   | Callback now receives the active index                                                                    |
| `tabSelect` on DBTabs   |   🔁   |   `indexChange`    | Angular/Vue/WC event, now receives the active index                                                       |
| `alignment` on DBTabs   |   🔁   | `tabItemAlignment` | Renamed for clarity                                                                                       |
| `width` on DBTabs       |   🔁   |   `tabItemWidth`   | Renamed for clarity                                                                                       |
| `scrollLeftText`        |   🔁   | `scrollStartLabel` | Renamed — logical direction (start/end) for correct RTL support                                           |
| `scrollRightText`       |   🔁   |  `scrollEndLabel`  | Renamed — logical direction (start/end) for correct RTL support                                           |

### Structural HTML changes

- Tab items: `<label><input type="radio" ...></label>` → `<button role="tab" type="button">`
- Tab panels: `<section>` → `<div role="tabpanel">`
- Tab list: `<ul>` wrapper removed — `<div role="tablist">` is now the direct container

### Behavior changes

<!-- markdownlint-disable MD033 -->

- **Tab panels are now keyboard focusable.** The active `tabpanel` receives `tabindex="0"` (inactive panels get `tabindex="-1"`), following the [WAI-ARIA APG tabs pattern](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/) so keyboard users can reach panel content that has no focusable element of its own. Previously the panel (`<section role="tabpanel">`) was not part of the tab order. If you relied on the old behavior, note that pressing <kbd>Tab</kbd> after the tab list now stops on the active panel.

<!-- markdownlint-enable MD033 -->

### Removed type exports

- **`OverflowScrollButtonProps`** has been removed from `@db-ux/core-components` shared model. Its props have been moved into `DBTabsProps` directly and renamed: `scrollLeftText` → `scrollStartLabel`, `scrollRightText` → `scrollEndLabel` (see table above). `arrowScrollDistance` keeps its name.
- **`OverflowScrollButtonState`** has been removed from `@db-ux/core-components` shared model. The scroll state is now internal to DBTabs.

## Removed default `margin-inline` from `[data-icon]` pseudo-elements

Previously, all `[data-icon]` / `[data-icon-leading]` elements had a default `margin-inline-end` on their `::before` pseudo-element, and `[data-icon-trailing]` elements had a default `margin-inline-start` on their `::after` pseudo-element. This default spacing (`var(--db-spacing-fixed-xs)`) has been removed.

**Why:** The default margin conflicted with the preferred approach of using `gap` on flex/grid containers. Every component using `gap` had to explicitly reset the margin to `0`, which was error-prone and added unnecessary overrides.

### What changed

| Before                                                                                                           | After                                      |
| ---------------------------------------------------------------------------------------------------------------- | ------------------------------------------ |
| `[data-icon]::before` had `margin-inline-end: var(--db-icon-margin-end, var(--db-spacing-fixed-xs))`             | No margin applied                          |
| `[data-icon-trailing]::after` had `margin-inline-start: var(--db-icon-margin-start, var(--db-spacing-fixed-xs))` | No margin applied                          |
| CSS custom properties `--db-icon-margin-end` and `--db-icon-margin-start` controlled the spacing                 | These custom properties are no longer used |
| SCSS variables `$default-icon-margin-end` and `$default-icon-margin-start` were available                        | These variables have been removed          |

### How to migrate

If you relied on the automatic icon margin for spacing between icons and text:

1. **Preferred: Use `gap` on the parent container** (recommended approach):

    ```css
    .my-component {
    	display: flex;
    	align-items: center;
    	gap: var(--db-spacing-fixed-xs); /* or your desired spacing */
    }
    ```

2. **Alternative: Add explicit margin to the icon pseudo-element**:

    ```css
    .my-component::before {
    	margin-inline-end: var(--db-spacing-fixed-xs);
    }
    ```

### Removed CSS custom properties

The following CSS custom properties are no longer functional:

- `--db-icon-margin-end` — previously controlled `margin-inline-end` on `::before`
- `--db-icon-margin-start` — previously controlled `margin-inline-start` on `::after`

If you were using these custom properties to adjust icon spacing, replace them with direct `margin-inline-end` / `margin-inline-start` on the respective pseudo-element, or use `gap` on the parent.

### Removed SCSS variables

- `icons.$default-icon-margin-end`
- `icons.$default-icon-margin-start`

If you referenced these in `calc()` expressions (e.g., for padding calculations), remove them and adjust your calculations accordingly.
