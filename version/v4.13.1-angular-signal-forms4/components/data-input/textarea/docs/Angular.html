How to use with Signal Formshow-to-use-with-signal-forms anchor link

Angular Signal Forms (Angular ≥ 21) are supported via Duck-Typing. The DBTextarea component exposes a value ModelSignal that Angular's [formField] directive recognizes automatically.

//app.component.ts
import { DBTextarea } from '@db-ux/ngx-core-components';
import { FormField } from '@angular/forms/signals';

@Component({
	// ...
	imports: [
		// ...,
		DBTextarea,
		FormField
	],
	// ...
})

// form.component.ts
import { Component, signal } from "@angular/core";
import { form } from "@angular/forms/signals";

export class FormComponent {
	model = signal({ message: "" });
	myForm = form(this.model);

	onFormSubmit(): void {
		alert(JSON.stringify(this.model()));
	}
}

<!-- form.component.html -->
<form (submit)="onFormSubmit()">
	<db-textarea
		label="Message"
		placeholder="Placeholder"
		[formField]="myForm.message"
	></db-textarea>
	<button type="submit">Submit</button>
</form>

<h2>Output</h2>
<dl>
	<dt>textarea's value</dt>
	<dd>{{ myForm.message().value() || "No message set" }}</dd>
</dl>

How to use with Reactive Formshow-to-use-with-reactive-forms anchor link

Third party controls require a ControlValueAccessor to function with angular forms. Our textarea component implements this interface so you can use it like any other native element with reactive Forms:

//app.component.ts
import { ReactiveFormsModule } from '@angular/forms';

@Component({
	// ...
	imports: [
		// ...,
		ReactiveFormsModule
	],
	// ...
})

<!-- form.component.html -->
<form [formGroup]="form" (submit)="onFormSubmit()">
	<db-textarea
		label="Textarea"
		placeholder="Placeholder"
		formControlName="textarea"
	>
	</db-textarea>
</form>

<h2>Output</h2>
<dl>
	<dt>textarea's value</dt>
	<dd>
		{{ form.get("textarea")?.value ? form.get("textarea")?.value : "No value
		set" }}
	</dd>
</dl>

// form.component.ts
export class FormComponent {
	form = new FormGroup({
		textarea: new FormControl("Filled with formControl")
	});

	onFormSubmit(): void {
		alert(JSON.stringify(this.form.value));
	}
}

How to use with Template Driven Formshow-to-use-with-template-driven-forms anchor link

Third party controls require a ControlValueAccessor to function with angular forms. Our textarea component implements this interface so you can use it like any other native element with ngModel:

//app.component.ts
import { FormsModule } from '@angular/forms';

@Component({
	// ...
	imports: [
		// ...,
		FormsModule
	],
	// ...
})

<!-- form.component.html -->
<form>
	<DBTextarea
		ngDefaultControl
		[(ngModel)]="textarea"
		label="Textarea"
		placeholder="Placeholder"
	></DBTextarea>
	<DBButton type="button" variant="brand" (click)="showValues()"
		>Get textarea value</DBButton
	>
</form>

<h2>Output</h2>
<dl>
	<dt>textarea's value</dt>
	<dd>{{ textarea ? textarea : "No value set" }}</dd>
</dl>

// form.component.ts
export class FormComponent {
	textarea = "";
	showValues(): void {
		alert(JSON.stringify({ textarea: this.textarea }));
	}
}
